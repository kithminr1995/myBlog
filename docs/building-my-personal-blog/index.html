Building my personal blog ‚úçÔ∏è

Engineering • Jun 02, 2020

How things got to where they are now...

I have been interested in creating a personal blog since I was in my second year at University, back in 2017. I gave it a try by making it on wordpress once, but it didn't feel personal enough. I realized that I didn't just want to have my own blog, I wanted to create it myself personally, like I engineered it with a bit of my lazy computer programming skills. So until then, I collected all the documents and images of my projects and endeavors and waited for the opportunity to do this.

I was able to get the exact opportunity to re-attempt this in 2020 when I saw the blog of a young undergraduate from the 17 batch; Ramith Hettiarachchi, who also happens to be from the SEF ScholarX Class of 2019.

Ramith had a very attractive blog, which looked very personal and well developed. I would like to extend my personal thanks to Ramith for his advice and support (and especially on how he got me out of the Netlify trouble which could have ended up in this particular article being entirely different to what it is now), in helping me write about my experience on "HOW I MADE MY ¬†SELF-HOSTED PERSONAL BLOG USING GHOST AND GITHUB PAGES".

PLEASE NOTE: I did everything on my Laptop which was running Windows 10. I didn't dual boot my laptop to get Linux thanks to the amazing work they have done with Windows Subsystem for Linux ¬†(WSL). Also, I did everything for FREE, which is how I like to work on projects. >Great results for Low Budget! üòâ

What I wanted to make here?

I was very excited about the year 2020, even though (so far) it was one of the most terrible years in Human History. This was solely due to the fact that 2020 opened up new avenues for me (and Netflix said a Narnia remake is coming soon üòç), in various fields such as Artificial Intelligence, Blogging and Product Development Research to name a few. I wanted to pick up some new skills I could use in my future career as well.

Ramith had used this API called Ghost, which was a powerful and unique tool for publishing online content. Together with Gatsby, I believed that it gave the complete FRONT-END framework I required for my custom blog.

The entire blog had to be hosted on github, and then I could build and deploy my new project with Netlify, and it will be hosted on the internet like any other Blog.

NOTE: This was how I envisioned everything to be developed at first. However the Netlify integration did not work out the way it was supposed to, and me and Ramith spent some time looking for other alternative methods. So this post is actually taking one such alternative approach and avoiding a whole lot of mess and time-waste. You're welcome üòÑ

Anyways, when I started to create this new blog, I found out that these websites here are very influential for my work. (Ignore the sections on setting up Netlify)

PLEASE NOTE: They used to do this kind of static blog creation using a package called buster, but now it is outdated and not available for Ubuntu 20.04 LTS. Our approach will go with a different static site generator tool instead of Buster! Much simpler!

Install Ubuntu 20.04 on Windows 10 using WSL

This was the big jewel that made this all possible. Thanks to the Windows Subsystem for Linux (WSL), I was able to get this project up and running like a charm.

This website https://itsfoss.com/install-bash-on-windows/ had the steps on how to install WSL on my computer. It was so simple!

PLEASE NOTE: Make sure to RESTART the computer when you install Ubuntu 20.04 LTS on Windows 10.

Now if you have Ubuntu running, ¬†add your user name and your password and get into your Linux terminal. 

sudo apt-get update && sudo apt-get upgrade

OPTIONAL: https://docs.microsoft.com/en-us/windows/wsl/wsl2-install

There is a small issue in WSL that you need to correct using the following code snippet. ¬†

sudo add-apt-repository ppa:rafaeldtinoco/lp1871129
sudo apt update
sudo apt install libc6=2.31-0ubuntu8+lp1871129~1 libc6-dev=2.31-0ubuntu8+lp1871129~1 libc-dev-bin=2.31-0ubuntu8+lp1871129~1 -y --allow-downgrades
sudo apt-mark hold libc6

Install yarn

This is what I installed first. Simply followed the instructions given here. 

curl -sS https://dl.yarnpkg.com/debian/pubkey.gpg | sudo apt-key add -
echo "deb https://dl.yarnpkg.com/debian/ stable main" | sudo tee /etc/apt/sources.list.d/yarn.list
sudo apt update && sudo apt install yarn
yarn --version

Install VS Code and link to WSL: Ubuntu 20.04

This is really elegant! To install VS Code on Linux and to be able to open it on your main desktop VS Code application. You can even access the file explorer, which shows you entire Linux filesystem. Its basically having the best of Windows and Linux together in a way.

mkdir <folder_of_blog>
cd <folder_of_blog>
code .

Install Gatsby-CLI + setup gatsby-starter-ghost

Now we can move into the really interesting section, how to get everything set up.

sudo yarn global add gatsby-cli
git clone https://github.com/TryGhost/gatsby-starter-ghost.git
cd gatsby-starter-ghost
sudo yarn install
sudo gatsby develop
