Install Ubuntu 20.04 on Windows 10 using WSL

This was the big jewel that made this all possible. Thanks to the Windows Subsystem for Linux (WSL), I was able to get this project up and running like a charm.

This website https://itsfoss.com/install-bash-on-windows/ had the steps on how to install WSL on my computer. It was so simple!

PLEASE NOTE: Make sure to RESTART the computer when you install Ubuntu 20.04 LTS on Windows 10.

Now if you have Ubuntu running, ¬†add your user name and your password and get into your Linux terminal. 

sudo apt-get update && sudo apt-get upgrade

OPTIONAL: https://docs.microsoft.com/en-us/windows/wsl/wsl2-install

Install Gatsby-CLI + setup gatsby-starter-ghost

Now we can move into the really interesting section, how to get everything set up using the WSL that you have just got up and running. It should be stated here that you don't really need to have gatsby installed for this website to to be deployed, since we wont be using the netlify integration which was part of the old approach I was trying to figure out. In this simpler method, doing the following is enough.

git clone https://github.com/TryGhost/gatsby-starter-ghost.git
cd gatsby-starter-ghost

Just in case you are interested, here is the setup installing the Gatsby-CLI . But you can IGNORE this part since you don't need it and move to the next topic.

sudo yarn global add gatsby-cli
git clone https://github.com/TryGhost/gatsby-starter-ghost.git
cd gatsby-starter-ghost
sudo yarn install
sudo gatsby develop

Now go to http://localhost:8000/ and you will see the following site. (localhost implies to the IP address 127.0.0.1 which belongs to your local machine). You will find this same website at https://demo.ghost.io/.

Deploy the project on a GitHub repository

Once you have a cloned version of the gatsby-starter-ghost repository, you need to push this repository to your own GitHub repository. To do that you will have to first create your own GitHub repository (Assuming that you already have a GitHub account here)

Now follow the commands below on your terminal inside gatsby-starter-ghost.

# Remove the original remote repository URL to gatsby-starter-ghost.git
git remote remove origin

# Add a new remote repository called origin, which points to your GitHub repo
# For eg- I did: git remote add origin https://github.com/kithminr1995/myBlog
git remote add origin YOUR_GIT_URL

# Push the code on your local machine to the GitHub repository
git push -u origin master

You will need to adjust few more setting in GitHub here and there, nothing too big. If the last command from above does not work, git push -u origin master --force should work.

Install Ghost and get the CMS running locally

After you have the gastby-starter-ghost pushed to your own repo, follow the steps below to create your own ghost_admin folder for this CMS (content management system) and install ghost locally on your computer.

cd ..
sudo yarn global add ghost-cli@latest
mkdir ghost_admin
cd ghost_admin
ghost install local

# Ghost will be started now and will be hosted at 127.0.0.1:2368
ghost stop
ghost start

You can access the admin interface at: http://localhost:2368/ghost/#/site. So simply go to this site, or copy and paste the link on your terminal. NOTE - Even if you close this terminal, ghost will still be running, only ghost stop command will stop the locally hosted admin interface.

Once you have finished the registration, you will be directed to

So just go through some of the properties and abs here and get an idea on how things work. Try to make some changes to the published contents and also make some changes to the themes. Also look into integrations, which are pretty cool!

You will observe that the changes you make will be available here at the, a locally hosted version of your blog website.

SITE: http://localhost:2368

Generate the static files to /docs and publish

If you can see all your changes and images you added on the locally hosted blog, then comes the really important part. It is time to generate the files containing a static copy of your locally hosted blog, which will be published to GitHub. We will simply use the wget command for this and publish all the files into a folder called docs.

cd gatsby-starter-ghost
wget -r -nH -P docs -E -T 2 -np -k http://localhost:2368/

The wget command will crawl the blog that is published at http://localhost:2368/ and create a static copy of that inside the docs directory. You can open the file docs/index.html in your web browser to verify. NOTE - So for this to work, you must make sure your ghost CMS is started and your blog is locally published.

Simply Push to GitHub and set it up

Now we have come to the last step. Hope you are excited to see the results of your're effort! üòÑ Follow the commands given below, and then go to your GitHub project repository.

NOTE: Use the following code snippet given below every time you make any updates and want to push to the GitHub repo to publish those changes.
wget -r -nH -P docs -E -T 2 -np -k http://localhost:2368/
git add -A
git commit -am 'Update to static website'
git push

Now all we need to do, is to enable GitHub Pages on our repository, so that the contents under docs gets published. For that, go to the "Settings" tab in your repository, and scroll down until you find the GitHub Pages section and choose the master branch /docs folder option and click the "Save" button:

We are done! After a few minutes (usually takes 2-5 minutes for the contents to be published the first time, afterwards updates are nearly instantaneous), you will can visit http://<github-username>.github.io/<github-repo-name>, and you will find your blog website published there.
